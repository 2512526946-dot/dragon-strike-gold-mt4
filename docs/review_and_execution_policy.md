# 观察信号、纸面复盘与人工执行政策

本文定义观察信号、纸面复盘、人工执行记录和仓位设计原则。当前阶段不做真实交易策略、不做风控计算、不做仓位计算。

## 观察信号与纸面复盘

短期阶段，用户不会真实下单。巨龙出击可以生成买卖点提示，但这些必须定义为：

- 观察信号
- 模拟买点
- 模拟卖点
- 参考入场区
- 信号候选
- 纸面复盘信号

这些信号必须标记为 `observation_only`。

不得定义为：

- 真实买入
- 真实卖出
- 实盘建议
- 已交易信号

用户未真实下单时：

```text
execution_mode = observation_only
executed_by_user = false
real_pnl = null
```

可以记录：

- `paper_result`
- `paper_mfe`
- `paper_mae`
- `paper_time_to_result`
- `review_label`

重要边界：

- 系统信号结果不等于真实交易盈亏。
- 纸面盈利不等于用户真实盈利。
- 未执行信号不得计入真实 PnL。

## 人工下单与执行复盘

巨龙出击未来必须区分三件事：

1. 系统机会
2. 用户人工决策
3. 实际 MT4 执行

短期用户可以手动标记：

- 我已执行
- 我未执行
- 我放弃了
- 我晚进场了
- 我提前进场了
- 我没有等确认

中期通过 MT4 只读导出成交记录：

- `symbol`
- `order_type`
- `open_time`
- `open_price`
- `lot`
- `stop_loss`
- `take_profit`
- `close_time`
- `close_price`
- `profit`
- `commission`
- `swap`
- `comment`

长期通过 `signal_id` 自动匹配系统机会和真实交易。

复盘分类：

- 系统给机会，用户执行了：复盘真实交易结果。
- 系统给机会，用户没执行：只复盘信号质量，不计真实盈亏。
- 系统禁止交易，用户却执行了：纪律违规。
- 系统没给机会，用户自己执行了：计划外交易。
- 系统建议小仓，用户实际重仓：仓位违规。
- 系统建议止损，用户没有止损：风控违规。

该功能只用于复盘和纪律分析，不用于自动下单。

## 仓位设计原则

未来巨龙出击必须具备仓位建议功能，但仓位模块不能在当前阶段开发。

仓位不是固定手数，而是根据以下因素动态计算：

- 账户权益
- 止损距离
- `tick_size`
- `tick_value`
- `min_lot`
- `lot_step`
- `spread`
- `slippage`
- 风险规则

原则：

- 单笔最大亏损限制为账户权益的 1%。
- 单日最大亏损限制约为账户权益的 3%。
- 如果最小手数下风险超过 1%，即使方向有机会，也必须禁止开仓。
- 如果单日亏损接近或达到 3%，当天禁止新交易信号。
- 不允许隔夜。
- 10x 不是每次打满，而是压力测试模式。
- 智能体可以解释仓位建议，但不能绕过仓位规则。
- 机器学习未来可以辅助过滤信号质量，但不能一开始控制仓位。

未来每次信号必须记录：

- 建议手数
- 风险金额
- 风险比例
- 杠杆状态
- 是否允许开仓
- 禁止原因

未来仓位输出不应只显示建议手数，还应显示预计亏损金额、预计亏损比例、止损距离、点差成本、最小手数风险、10x 压力测试、是否允许人工决策和禁止原因。

## 当前阶段边界

当前阶段只保留这些政策作为架构决策，不实现仓位计算、不实现风控计算、不输出真实交易建议。

## 模拟账号训练补充边界

未来 MT4 模拟账号训练可以分为只读、半自动模拟交易、极小手数自动模拟训练和长期模拟验证四个阶段。

半自动阶段中，智能体只能生成 TradePlan，不得直接下单。用户确认不是风控豁免，确认后仍必须通过 DataQualityGate、RiskGate、PositionSizing 和 ExecutionGate。

没有止损价、没有账户净值、数据过期、风控不通过、非 demo 账户，均不得执行。未来自动模拟训练默认关闭，不允许马丁加仓、不允许网格加仓、不允许隔夜持仓。
