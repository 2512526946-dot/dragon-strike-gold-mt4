# Demo Execution Authority Principles

本文定义巨龙出击未来 Demo 执行链路的权限原则。本文只是一份架构原则文档，不实现 Agent、LLM、RiskGate、PositionSizing、ExecutionGate、TradePlanSchema、EA、API、Dashboard、MT4 接入或任何下单能力。

当前项目仍处于只读副驾驶大阶段。所有当前能力都必须保持 demo-only、read-only、安全解释、只读诊断和文档规划边界。实盘交易不属于当前阶段，不得实现。

## 最高原则

巨龙出击的未来 Demo 执行权限必须遵守以下最高原则：

智能体只有建议权，程序闸门有否决权，EA 才有执行权。

- 智能体只有建议权。
- 程序闸门拥有否决权。
- EA 只有执行已授权订单的权限。
- 半自动阶段用户拥有最终确认权。
- 自动模拟训练默认关闭。
- 实盘交易不进入当前阶段。

这些原则的含义是：Agent / LLM 可以分析、解释和生成草案，但不能直接执行；DataQualityGate、RiskGate、PositionSizing 和 ExecutionGate 可以阻断；MT4 EA 只能执行已经被用户确认或被后期自动模拟训练模式授权、并且通过全部程序检查的 Demo 订单。

## 未来目标执行链路

未来如果进入 Demo 执行训练阶段，目标链路必须是：

```text
智能体生成交易计划
  -> DataQualityGate 检查数据是否可靠
  -> RiskGate 检查是否允许交易
  -> PositionSizing 根据账户净值、止损距离、风险比例和黄金合约参数硬计算手数
  -> ExecutionGate 检查是否允许执行
  -> 用户确认 / 或后期自动模拟训练模式开启
  -> MT4 EA 在 Demo 账号执行
  -> 记录交易日志、审计日志和复盘数据
```

链路规则：

- 智能体不能绕过任何 Gate。
- 智能体不能直接调用 EA。
- 智能体不能修改风控规则。
- PositionSizing 只能按硬规则计算。
- PositionSizing 不接受 Agent 自由给定 lot。
- ExecutionGate 可以否决。
- EA 只能执行已通过全部检查的订单。
- 任意 Gate 拒绝后不得继续执行。
- source ready、diagnostics passed、explanation ready 或 dashboard healthy 都不是交易许可。

## 权限矩阵

| 角色 | 可以做 | 不可以做 |
| --- | --- | --- |
| Agent / LLM | 分析市场上下文；解释只读诊断；生成交易计划草案；说明风险和不确定性 | 下单；修改风控；调用 EA；生成最终手数；覆盖任何 Gate；开启自动模拟训练；连接 MT4 或账户 |
| DataQualityGate | 检查行情、K 线、品种规格、账户快照等数据是否可靠；否决数据不可靠的计划 | 放宽交易权限；生成行情观点；给出执行许可 |
| RiskGate | 按风控规则否决交易；检查单笔风险、日亏损、连续亏损、交易次数、重大数据窗口、隔夜限制 | 被 Agent 覆盖；放宽硬风控；生成仓位或行情观点 |
| PositionSizing | 使用账户净值、止损距离、风险比例、黄金合约参数硬计算手数；限制最大风险 | 接受 Agent 自由给定 lot；超过最大风险限制；在合约参数不可信时继续计算可执行手数 |
| ExecutionGate | 最终检查是否具备执行条件；确认止损、风险、数据新鲜度、账户模式、授权状态；否决执行 | 生成行情观点；修改风控规则；扩大手数；替代用户确认 |
| User | 半自动阶段手动确认；取消、暂停或关闭训练模式；人工接管 | 用确认动作绕过程序 Gate；把只读输出视为实盘许可 |
| MT4 EA | 只执行通过授权的 Demo 订单；回传执行结果和审计信息 | 分析行情；扩大手数；取消止损；马丁；网格；隔夜持仓；接收未授权命令；连接实盘执行 |

## 阶段划分

### 第一阶段：只读副驾驶

当前项目仍处于这个大阶段。

边界：

- 只读。
- 不下单。
- 不生成可执行订单。
- 不调用 EA。
- 不连接模拟账号或真实账号进行执行。
- 不把只读数据、诊断、解释或文档规划解释为交易许可。

允许的工作包括：文档规划、只读文件格式、只读 reader、只读 diagnostics、只读 explanation、前端只读展示、安全回归测试和 demo-only/read-only 合同校验。

### 第二阶段：半自动模拟下单

未来只有满足前置条件后才可进入，且只能用于 Demo 账号。

边界：

- 只能 Demo 账号。
- 必须用户确认。
- 必须有止损。
- 必须通过 DataQualityGate、RiskGate、PositionSizing 和 ExecutionGate。
- EA 才能执行。
- 用户确认不是风控豁免。
- 智能体仍然只有建议权。
- 程序 Gate 仍然拥有否决权。
- 实盘交易不属于本阶段。

### 第三阶段：自动模拟训练

未来自动模拟训练必须默认关闭，并且只允许 Demo。

边界：

- 默认关闭。
- 只允许 Demo。
- 只允许小手数，初始建议 0.01。
- 必须强制止损。
- 必须每日次数限制。
- 必须连续亏损停手。
- 必须日亏损停手。
- 必须重大数据窗口降级或禁开仓。
- 必须禁止隔夜。
- 必须记录审计日志。
- 必须允许用户随时暂停、关闭和人工接管。
- 不得作为实盘自动交易功能。

实盘交易不属于当前阶段，不得实现。

## 硬性风控禁止项

以下禁止项必须作为未来 Demo 执行链路的硬边界：

- 无止损禁止交易。
- 马丁加仓禁止。
- 网格加仓禁止。
- 隔夜持仓禁止。
- 重大经济数据前后必须降级或禁止开新仓。
- spread 过大禁止开仓。
- 数据延迟过大禁止开仓。
- 合约参数缺失禁止开仓。
- 点值、合约大小、最小手数不可信禁止开仓。
- 当日亏损达到阈值停手。
- 连续亏损达到阈值停手。
- 当日交易次数达到上限停手。
- 智能体高置信度不能覆盖程序风控。
- 任何 Gate 拒绝后不得继续执行。
- Agent / LLM 不得自行开启自动模拟训练。
- EA 不得扩大手数、取消止损、修改风控或接收未授权命令。
